Level A — Foundations

Brand Identity

Orion's brand is built on precision and clarity. The identity uses a minimal black-and-yellow palette drawn from the original Orion Job SaaS case study, adapted here for a data-dense IT strategy context. The result is a system that feels authoritative without sacrificing warmth.

Logo Mark

The Orion mark combines the O + R letterforms into a connective symbol representing the link between users and data. In the IT Strategy Dashboard, it appears in the sidebar and top bar only. Never recreate it in SVG — always use the supplied asset.

Primary on dark
Mono on dark
Mono on light

Clear-space & Misuse

Do
Maintain 16px clear-space on all sides. Use on black, white, or brand-yellow backgrounds only.
Don't
Place the mark on photos, gradients, or patterned surfaces. Do not recolor the mark.

Color System

Motion

Motion communicates state and hierarchy. Always animate transform and opacity — never layout properties. Use exponential easing for natural deceleration. Hover over the cards to preview each easing curve.

ease-out-quart
cubic-bezier(0.25, 1, 0.5, 1)
Default — buttons, menus, badges
ease-out-expo
cubic-bezier(0.16, 1, 0.3, 1)
Snappy — sheet slides, modals, toasts
ease-in-out
Built-in CSS
Smooth — progress bars, number counts
/* Duration tokens */ --dur-fast: 120ms; /* micro-interactions */ --dur-base: 200ms; /* button hover, tab switch */ --dur-slow: 350ms; /* panel slide, chart reveal */ /* Always use transform + opacity */ .card:hover { transform: translateY(-2px); box-shadow: var(--shadow-md); transition: transform var(--dur-base) var(--ease-out-quart), box-shadow var(--dur-base) var(--ease-out-quart); } /* Respect prefers-reduced-motion */ @media (prefers-reduced-motion: reduce) { * { animation-duration: 1ms !important; transition-duration: 1ms !important; } }

shadcn/ui Visual Mapping

Visual guidance for mapping Orion design tokens to shadcn/ui components. Shadcn uses CSS variables via --background, --foreground, --primary etc. Override these in your globals.css to apply the Orion theme. Actual component code mapping can be added in a later phase.

Orion Component shadcn/ui Component Key Token Overrides Notes
Button Primary Button variant="default" --primary: #F5C842; --primary-foreground: #111110 Yellow bg, black text. Override default blue.
Button Secondary Button variant="outline" --border: rgba(28,28,26,0.14) Use default outline with Orion border token.
Button Ghost Button variant="ghost" --accent: #F1F0EB Maps directly to shadcn ghost variant.
Input Input --ring: #F5C842; --input: rgba(28,28,26,0.14) Yellow focus ring. Override --ring token.
Card Card --card: #FFFFFF; --card-foreground: #111110; --radius: 12px Increase radius to 12px (xl) from shadcn default.
Badge Badge Custom variants per semantic color Extend shadcn Badge with success/warning/danger/info variants.
Tabs Tabs --primary: #F5C842 for active underline Underline style. shadcn default is pill — switch to trigger style override.
Data Table DataTable (TanStack) --muted: #F9F8F5; --border: rgba(28,28,26,0.08) Subtle row hover, thin borders.

LLM Design Rules

When prompting an LLM to generate UI for this dashboard, include these rules verbatim. They act as a constraint set that ensures generated output is on-brand, avoids common AI slop patterns, and produces production-ready code.

/** * ORION IT STRATEGY DASHBOARD — LLM DESIGN RULES * Include these in any AI system prompt generating UI for this product. */ // ── IDENTITY ──────────────────────────────────────────────────────────── PRODUCT: "IT Strategy Dashboard" BRAND: "Orion — precision, clarity, minimal black-and-yellow palette" FONT: "DM Sans for UI; DM Mono for data values only" ACCENT: "#F5C842 (yellow) — one accent, used for CTAs and active states ONLY" // ── COLOR RULES ────────────────────────────────────────────────────────── ✅ Use CSS custom properties: var(--orion-yellow), var(--orion-text-primary), etc. ✅ Background hierarchy: surface-0 (white) > surface-1 (#F9F8F5) > surface-2 (#F1F0EB) ✅ Sidebar always uses --orion-black (#111110) background ✅ Semantic color for status: success=#2D7A4F, warning=#B8700A, danger=#C33027, info=#1A5FA8 ❌ NO gradient text — especially on metrics or headings ❌ NO cyan-on-dark, purple-to-blue gradients, neon glows ❌ NO pure black (#000) or pure white (#fff) — use --orion-black and white tinted surfaces ❌ NO per-component hardcoded hex — always reference tokens // ── TYPOGRAPHY RULES ───────────────────────────────────────────────────── ✅ Display/H1 use clamp() for fluid sizing ✅ Data values: font-family: var(--font-mono) ✅ Weight: 400 regular, 500 medium, 600 semibold — never 700+ for body ✅ Letter spacing: -0.03em display, -0.02em H1, -0.01em H2/H3 ❌ NO Inter, Roboto, Arial as primary typeface ❌ NO font-size below 12px on any rendered text ❌ NO uppercase for body text — uppercase only for labels/micro-copy // ── LAYOUT RULES ───────────────────────────────────────────────────────── ✅ 12-column grid, 24px gutters, fluid margins (clamp 16px–48px) ✅ Dashboard: grid-template-columns: 220px 1fr with 56px top bar ✅ Stats row: 4 equal columns at ≥1024px, 2 at ≤768px ✅ Cards: border-radius 12px (var(--radius-xl)), 1px border-subtle, shadow-xs ✅ Spacing from scale ONLY: 4, 8, 12, 16, 20, 24, 32, 40, 48, 64px ❌ NO cards nested inside cards ❌ NO identical card grids (same icon + heading + text repeated) ❌ NO center-aligning everything — prefer left-aligned content ❌ NO glassmorphism, frosted glass, blur effects // ── COMPONENT RULES ────────────────────────────────────────────────────── ✅ Buttons: 36px default height, 1.5px border, border-radius 8px ✅ Primary button: yellow bg (#F5C842) with black text (#111110) ✅ Inputs: 40px height, yellow focus ring (3px, 20% opacity) ✅ Badges: pill radius, 12px font, semantic colors, dot variant for live status ✅ Progress bars: 6px height, pill radius ✅ Tables: 0px inner spacing between rows, 1px border-subtle separator ❌ NO modals for non-critical confirmation — use inline or sheet instead ❌ NO more than 3 badge variants in a single table row ❌ NO bounce/elastic animation easing // ── MOTION RULES ───────────────────────────────────────────────────────── ✅ Animate transform and opacity ONLY ✅ Duration: 120ms micro, 200ms interactions, 350ms panels ✅ Easing: cubic-bezier(0.25, 1, 0.5, 1) default, cubic-bezier(0.16, 1, 0.3, 1) for sheets ✅ Wrap all animations in @media (prefers-reduced-motion: no-preference) ❌ NO layout property animation (width, height, padding, margin) ❌ NO bounce or elastic easings // ── ANTI-PATTERNS (AI SLOP) ─────────────────────────────────────────────── ❌ Rounded rectangles with thick single-side colored border accent ❌ Hero metric layout: big number → small label → supporting stats → gradient accent ❌ Sparklines used as decoration (no interactable meaning) ❌ Generic icon + heading + body card grid repeated 3–6 times ❌ "dark mode with glowing neon accents" aesthetic ❌ Large emoji as section icons ❌ Excessive bolding mid-sentence for emphasis
